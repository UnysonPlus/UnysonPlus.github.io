A lot has shipped across the Unyson+ plugin and theme recently. Because everything updates straight
from GitHub, you already have these, here's what's worth knowing about.

<!-- truncate -->

## Theme Builder

The biggest addition: a **[Theme Builder](/docs/extensions/theme-builder)**, our take on Divi's. Build
global **headers**, **footers**, and full-page **body** layouts with the visual builder, bundle them
into a **Template**, and assign each Template to parts of your site with conditional "Use On / Exclude
From" rules. It installs from its own repo (a Download card in the Extensions manager) and absorbs the
old header/footer presets.

New here? The [Build a sticky header](/docs/guides/sticky-header) and
[custom 404 page](/docs/guides/custom-404-page) guides are good starting points.

## Nested columns

A page-builder **column can now host other columns** (one level deep), not just leaf elements. Drag a
column onto the interior of another column to build richer grids, the items corrector synthesizes the
inner row automatically, so existing width fractions just work. See
[Column widths](/docs/page-builder/column-widths).

## Site Converter: Google Stitch import

The **[Site Converter](/docs/extensions/site-converter)** can now turn a **Google Stitch** design
straight into a native child theme + a page-builder Full Page, deterministically, with no AI step.
Stitch joins the existing media / presets / theme-settings / pages / menus pipeline and the one-shot
Convert bundle.

## Faster, sharper images

Images rendered through the framework now ship a responsive `srcset`/`sizes`, explicit
`width`/`height` (for zero layout shift), and smart `fetchpriority`/lazy loading, so pages stay fast
and crisp on every screen. More in [Performance](/docs/performance).

## Better forms & options

Option and form fields gained **inline validation** in the editor plus an opt-in **server-side
validation** layer, so bad input is caught with a clear, on-brand message instead of a silent save.

## Keeping up to date

The plugin, theme, and every extension update from the WordPress dashboard, just keep them current.
For the full picture, the [manual](/docs/intro) now covers every feature, with
[Guides](/docs/guides) for common tasks and a [Troubleshooting](/docs/troubleshooting) page when
something doesn't go to plan.
