# Header & Footer Builder

Design headers and footers visually with the Page Builder instead of editing theme templates.
Each preset is a **Header** or **Footer** post built with the builder and rendered into a
semantic `<header>` / `<footer>`. Assign a preset per page/post or site-wide; the Theme
Settings header/footer remains the default when nothing is assigned.

## Typical workflow

1. Activate **Header & Footer Builder** from **Unyson+ → Extensions**.
2. Create a **Header** (or **Footer**) preset and build it on the canvas using the
   [Header / Footer Elements](/docs/shortcodes/header-footer-elements) — Site Logo, Navigation
   Menu, Site Search, Social Icons and Menu Toggle — alongside any other builder element.
3. Assign the preset **site-wide**, or override it on individual pages/posts.
4. The preset renders into a real semantic `<header>` / `<footer>` on the front end.

## Header / Footer elements

The builder gains a dedicated group of chrome elements that pull live data from WordPress:

- **Site Logo** — Site Identity logo/title or a custom image.
- **Navigation Menu** — any registered menu, as a dropdown, mega or accordion.
- **Site Search** — inline or expanding-icon search form.
- **Social Icons** — from Theme Settings social profiles or a manual list.
- **Menu Toggle** — a hamburger button that opens an off-canvas drawer.

See [Header / Footer Elements](/docs/shortcodes/header-footer-elements) for each element's
options.
